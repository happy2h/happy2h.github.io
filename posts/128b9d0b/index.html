2021-02-25-在hexo-next主题下添加相册模块

next主题美化
NEXT主题
发布日期:   2021-02-25
更新日期:   2021-03-07
文章字数:   1.5k
阅读时长:   6 分

起因

马上就要开学了，在家玩的也不开心，毕竟一整个暑假也没做多少与学习相关的东西，总感觉心很慌。家外面也在刮大风下雨，打游戏也打不进去，闲来无事就想把博客给折腾折腾，看到别的博主的帖子可以给NexT添加类似画廊一样的推文，就想着能不能在菜单添加一个相册模块。首先当然是Google一下，找到了几个相似的帖子，记录一下操作的流程和碰到的一些问题。

菜单栏开启相册

首先在next主题配置文件下搜索menu，新建一行

photograph: /photograph || fa fa-camera

然后在blog根目录下右键运行Git Bash，执行hexo new page photograph，注意这里的page名要与配置文件里的一致。执行完成以后可以在hexo/source下生成了一个photograph文件夹和里面的index.md。对index.md文件进行更改，添加以下代码：

---
title: 照片
date: 
type: photos
fancybox: false
comments: false
---
<link rel="stylesheet" href="../lib/album/ins.css">
<link rel="stylesheet" href="../lib/album/photoswipe.css"> 
<link rel="stylesheet" href="../lib/album/default-skin/default-skin.css"> 
<div class="photos-btn-wrap">
  <a class="photos-btn active" href="javascript:void(0)" target="_blank" rel="external">Photos</a>
</div>
<div class="instagram itemscope">
  <a href="http://yourbolg.com" target="_blank" class="open-ins">图片正在加载中…</a>
</div>
 
<script>
  (function() {
    var loadScript = function(path) {
      var $script = document.createElement('script')
      document.getElementsByTagName('body')[0].appendChild($script)
      $script.setAttribute('src', path)
    }
    setTimeout(function() {
        loadScript('../lib/album/ins.js')
    }, 0)
  })()
</script>

建立GitHub相册仓库

  • 首先在自己的账号下建立一个新仓库，复制仓库的http地址，然后在自己的电脑盘上任意位置新建一个存放芯片和py工具的文件夹，然后在该文件夹下右键运行Git Bash，打开后执行git init，然后执行git remote add origin https://github.com/yourname/repositoryname.git。
  • 然后在该文件夹下新建一个photos文件夹（注意这个文件夹名不能更改）。然后下载tool.py文件。注意更改一下tool.py文件里的json输出路径，打开tool文件搜索/source/lib/album/data.json。photos里的图片文件需要以20xx-01-01_description的方式命名，日期和描述之间是下划线。

  • 在blog/theme/next/source/lib下新建一个album文件夹。

  • 回到存有tool.py的文件夹下，在运行tool.py之前想把要展示的相片存放到photos文件夹下。然后在bash里运行python tool.py，如果提示缺少PIL库，则运行pip3 install Pillow，然后重新执行即可。

相册相关样式配置

  • 首先下载这个文件配置压缩包，解压缩以后，把JS&CSS文件夹中所有文件复制到theme/next/source/lib/album/中。

  • 将压缩包里的photoswipe.min.js和photoswipe-ui-default.min.js两个文件添加到theme/next/source/js/src/中。没有对应文件夹就自己建一个。

  • 更改album中的ins.js，更改文件的122行和125行的仓库地址。

-    var minSrc = 'https://raw.githubusercontent.com/lizhi/Blog_Album/master/min_photos/' + data.link[i];
-    var src = 'https://raw.githubusercontent.com/lizhi/Blog_Album/master/photos/' + data.link[i];
+    var minSrc = 'https://raw.githubusercontent.com/yougithubname/pathtoyourphotos/min_photos/' + data.link[i];
+    var src = 'https://raw.githubusercontent.com/yougithubname/pathtoyourphotos/photos/' + data.link[i];

注意这里图片的地址不是你仓库打开的地址，而是把仓库的图片在新标签页打开的地址，否则无法正常显示图片。

引用配置

  • 在next/layout/_layout.swig的head前添加对js文件的引用：

    <script src="{{ url_for(theme.js) }}/src/photoswipe.min.js?v={{ theme.version }}"></script>
<script src="{{ url_for(theme.js) }}/src/photoswipe-ui-default.min.js?v={{ theme.version }}"></script>

  • 在next/layout/_layout.swig文件，在body中添加：

    {% if page.type === "photos" %}
  <!-- Root element of PhotoSwipe. Must have class pswp. -->
  <div class="pswp" tabindex="-1" role="dialog" aria-hidden="true">
    <div class="pswp__bg"></div>
    <div class="pswp__scroll-wrap">
        <div class="pswp__container">
            <div class="pswp__item"></div>
            <div class="pswp__item"></div>
            <div class="pswp__item"></div>
        </div>
        <div class="pswp__ui pswp__ui--hidden">
            <div class="pswp__top-bar">
                <div class="pswp__counter"></div>
                <button class="pswp__button pswp__button--close" title="Close (Esc)"></button>
                <button class="pswp__button pswp__button--share" title="Share"></button>
                <button class="pswp__button pswp__button--fs" title="Toggle fullscreen"></button>
                <button class="pswp__button pswp__button--zoom" title="Zoom in/out"></button>
                <!-- Preloader demo http://codepen.io/dimsemenov/pen/yyBWoR -->
                <!-- element will get class pswp__preloader--active when preloader is running -->
                <div class="pswp__preloader">
                    <div class="pswp__preloader__icn">
                      <div class="pswp__preloader__cut">
                        <div class="pswp__preloader__donut"></div>
                      </div>
                    </div>
                </div>
            </div>
            <div class="pswp__share-modal pswp__share-modal--hidden pswp__single-tap">
                <div class="pswp__share-tooltip"></div> 
            </div>
            <button class="pswp__button pswp__button--arrow--left" title="Previous (arrow left)">
            </button>
            <button class="pswp__button pswp__button--arrow--right" title="Next (arrow right)">
            </button>
            <div class="pswp__caption">
                <div class="pswp__caption__center"></div>
            </div>
        </div>
    </div>
  </div>
{% endif %}

    然后可以hexo s看一下效果，细微地调整一下，然后部署即可。

遇到的问题及解决

  • 执行tool.py时会显示空，其实是photos文件夹里没有图片就执行了tool.py，看来心急吃不了热豆腐啊😂。

  • 改完以后图片不显示，检查出问题是出在ins.js的仓库地址的添加下，一定要以raw.githubusercontent开头的链接。

  • 样式显示太拥挤，这个我是改了一点，忘记截图改之前的了🤢​​。就是我的主题是Gemini，我上传了四张二月份的图片，但是flex容器没有换行显示，而且图片之间距离很近，需要搭配网页调试工具对ins.css进行修改。这是我改过的文件：

    .instagram .thumb {
	width: 25%;
	margin: 10px;
	background: #ededed;
	outline: 1px solid #ddd;
}
.posts-expand .post-body img {
	padding: 0;
    border: 0;
	box-sizing: border-box;
    margin: 0px;
}
.instagram .album ul{
	display: flex;	
	flex-wrap: wrap;
	clear: both;
	width: 100%;
	box-sizing: border-box;
	margin: 12px;
	padding: 5px;
}

    看看效果，还行​​😀
