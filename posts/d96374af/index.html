阿里云域名购买及配置

在把GitHub page和hexo主题搭建起来以后，就想让博客能够被搜到，这里记录一下，现在好像只有Google可以搜到。这里只是我的过程步骤，也是第一次配置，如有错误，希望可以指出我来更改。

sitemap的配置

这一步是必不可少的，这里介绍一下网上大家常用的方式：

  1. 执行安装命令
npm install hexo-generator-sitemap --save
npm install hexo-generator-baidu-sitemap --save
  1. 在hexo的站点配置文件里添加如下代码
plugins:
  - hexo-generator-sitemap
  - hexo-generator-baidu-sitemap
  # 自动生成sitemap
sitemap:
  path: sitemap.xml
baidusitemap:
  path: baidusitemap.xml
  1. 执行hexo cl和hexo g来生成sitemap.xml文件。可以在站点的public文件夹下看到生成两个xml文件。

阿里云购买和配置

域名配置

进入到阿里云，产品服务里选域名，选择购买二手域名：

购买以后在控制台里选择解析：

点击添加记录，选择CNAME，如图：

这里以后有两个步骤，我不清楚先后顺序，但是我先给出我的操作吧。

先去你的仓库地址里，在settings里面添加你刚刚购买的域名

成功的话应该会显示Your site is published at xxxx。

站点配置

在站点的_config.yml文件下将url的值改为你购买的域名：

我也把主题文件夹下的_config.yml中menu下sitemap值取消注释和baidu_push的值设置为true。执行hexo cl、hexo g和hexo -d提交到GitHub仓库。

这里可以测试一下，打开自己的github.io地址会不会跳转到你购买的域名。

提交Google和百度收录

百度收录

打开百度站长平台，点击添加网站，协议头选择https，网站填www加你的域名。然后等待他收录就行了。

我目前是到了https验证成功的这一步。

在验证网站这一步选择CNAME验证，就是在域名解析里添加一条CNAME记录，可以参考域名解析

Google收录

这个比较容易，打开Google Search Console，添加网域资源，里面添加你购买的域名：

这里也选择TXT文件验证，与CNAME文件类似，在阿里云控制台，点击域名解析，点击添加记录，记录类型选择TXT文件，主机记录选择@，记录值粘贴GSC复制的值，添加完成以后，回到Google点击完成验证，会显示验证成功。成功以后删除这一条记录，不然博客会访问不了。

验证完以后不用添加sitemap.xml文件，以后提交网域以后会自动提交以下的所有文件。

然后就是等待收录，会发一段时间，不用急。
